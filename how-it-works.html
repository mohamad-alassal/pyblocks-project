How PythonBlocks Works

A professional breakdown of every layer of the platform — from the visual editor to the Python execution engine running on the server.

Overview

What is PythonBlocks?

PythonBlocks is a visual programming environment built on top of Google's open-source Blockly library. It lets beginners and kids construct Python programs by snapping graphical blocks together — eliminating syntax errors and lowering the barrier to entry for real programming.

Every block in the workspace maps 1-to-1 to a Python statement or expression. As you build, a live code preview shows you the exact Python that will run. When you hit RUN_SCRIPT, that code is sent to a Node.js backend which executes it in a sandboxed Python process and streams the output back to you.

🧩

Block Editor

Drag-and-drop interface powered by Google Blockly. Every block is colour-coded by category.

⚙️

Code Generator

Blockly's Python code generator converts your workspace in real-time to valid Python 3 code.

🖥️

Server Execution

A Node.js Express server spawns a child Python process to safely run your code with a timeout.

💾

Project Storage

Projects are persisted in a local SQLite database via better-sqlite3 — no cloud needed.

Step by Step

Your Coding Journey

01

Create a Project

Go to the Projects page and click New Mission. Give your project a name and it gets saved immediately to the database. You can have as many projects as you want — each one is independent.

02

Open the Block Editor

Click a project card to open the visual editor. On the left you'll see the Toolbox — a categorised library of blocks covering Logic, Loops, Math, Text, Lists, Tkinter GUI, Variables, and Functions.

03

Drag & Drop Blocks

Click a category in the toolbox to reveal its blocks in a flyout panel. Drag any block onto the workspace canvas. Blocks snap together magnetically — output connectors on the right connect to input slots, and statement blocks stack top-to-bottom just like lines of code.

04

Watch Real Python Generate

The Code Preview panel at the bottom updates live as you edit. It uses Blockly's built-in Python code generator which understands operator precedence, indentation, and import statements. You can copy the code at any time with the Copy Code button.

05

Run Your Code

Click RUN_SCRIPT. The generated Python is sent via a POST /api/run request to the server. The server spawns python3 with a 5-second timeout. stdout and stderr are captured and returned to the Console Output panel so you see results instantly.

06

Auto-Save & Come Back

Every change triggers an auto-save after 1.5 seconds of inactivity. The workspace state (block positions and connections) is serialised to JSON and stored in the database. Reopening a project restores everything exactly as you left it.

Technology

What Powers This Platform

PythonBlocks is built entirely with open-source, well-maintained technologies. There are no frameworks like React or Vue — just vanilla HTML, CSS, and JavaScript on the front end, with a lightweight Node.js server on the back end.

Frontend

Google Blockly

An open-source JavaScript library from Google that provides the drag-and-drop block editor. It handles workspace rendering, block snapping, serialisation, and code generation for multiple languages including Python.

developers.google.com/blockly →
Frontend

Vanilla JS / CSS

All UI logic (dark mode, project management, code preview, console panel) is written in plain JavaScript with no dependencies. Styles use CSS custom properties (variables) for instant light/dark theming.

Backend

Node.js + Express

A small Express server exposes REST API endpoints: GET/POST/PUT/DELETE /api/projects for project management, and POST /api/run for code execution. It also serves all static files.

Backend

Python 3 Subprocess

When you run code, Node.js calls child_process.spawn('python3', ['-c', code]). The process runs with a hard 5-second timeout to prevent infinite loops from blocking the server. stdin can be pre-filled for input() calls.

Storage

SQLite via better-sqlite3

Projects are stored in a local SQLite database file. The better-sqlite3 package provides synchronous, high-performance access without needing a separate database server — perfect for a local development tool.

GUI Library

Tkinter Blocks

A custom set of Blockly blocks that generate valid tkinter Python code. This lets you build real desktop GUI windows — labels, buttons, entry fields, message boxes, and more — all by dragging blocks together.

FAQ

Common Questions

Is this safe for kids? ▼
Yes. Code runs in a separate Python child process that is killed after 5 seconds. It cannot access the operating system, the network, or any files outside the sandboxed environment beyond what Python's standard library provides.
Does Tkinter code actually open a GUI window? ▼
Tkinter generates code that is designed to run as a standalone Python script. Copy the generated code from the Code Preview panel, save it as a .py file, and run it with Python installed on your computer — the window will open. Running it inside the browser console won't show a GUI because there's no display available in the server subprocess.
Can I run code that uses input()? ▼
Yes. Type your input values (one per line) in the text area below the console before clicking RUN_SCRIPT. They are piped to the Python process's stdin, so each input() call will consume one line.
Where is my data stored? ▼
Everything is stored locally on your machine in a SQLite file (projects.db) inside the project folder. Nothing is sent to any external server or cloud service.
Can I see the Python code without running it? ▼
Absolutely. The Code Preview panel on the left side of the bottom bar shows your generated Python code live as you edit. Use the Copy Code button in the header to copy it to your clipboard.
What age group is this for? ▼
PythonBlocks is designed for kids aged 7 and up, and for any beginner who wants a visual introduction to Python before typing code. The Tkinter category is more suited to older students (11+) who want to start building real desktop apps.
